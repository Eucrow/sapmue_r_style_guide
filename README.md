# SAPMUE R Style Guide

This repository hosts the R style guide used within the Sampling Data Quality Assurance Team of the SAP project. It provides conventions and best practices for writing clear, consistent, and maintainable R code.

- Core document: see [`r_style_guide.md`](./r_style_guide.md)
- License: see [`LICENSE`](./LICENSE)

## Using the Guide

- Reference the guide during code reviews to align on decisions.
- Prefer consistent naming, formatting, and documented functions.
- When in doubt, default to the guidance in the style guide and document exceptions.

## Contributing

Improvements are welcome. To propose changes:

1. Open an issue describing the change and rationale.
2. Provide examples or references if relevant.
3. Submit a pull request updating `r_style_guide.md` with the proposed edits.

Changes should aim to improve clarity, developer experience, and consistency.
